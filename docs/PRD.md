# PRD — Product Requirements Document
# Kanban Board

---

## 1. 개요

### 1.1 목적
브라우저에서 즉시 사용할 수 있는 칸반보드. 별도 설치나 로그인 없이 작업 상태를 시각적으로 관리한다.

### 1.2 대상 사용자
- 개인 할 일 관리가 필요한 사용자
- 소규모 팀의 작업 흐름을 시각화하려는 팀

### 1.3 비즈니스 목표
- 작업 상태를 한눈에 파악하여 생산성 향상
- 드래그 앤 드롭으로 최소한의 조작으로 작업 관리
- 브라우저 새로고침 후에도 데이터 유지

## 2. 기능 요구사항

### 2.1 핵심 기능 (Must Have)

#### F-01. 3단계 상태 컬럼
- **To-do**: 시작 전 작업
- **In-progress**: 진행 중 작업
- **Done**: 완료된 작업
- 각 컬럼은 독립적으로 카드를 보유

#### F-02. 카드 추가
- 각 컬럼 헤더의 `+` 버튼으로 카드 생성
- 빈 내용의 카드는 생성 불가

#### F-03. 카드 삭제
- 데스크탑: 카드 호버 시 `×` 버튼 노출
- 모바일/터치: `×` 버튼 항상 표시
- 클릭 즉시 제거

#### F-04. 카드 이동 (드래그 앤 드롭)
- 데스크탑: 마우스 드래그로 컬럼 간 이동 (HTML5 Drag and Drop API)
- 모바일/터치: 터치 드래그로 컬럼 간 이동 (Touch Events API)
- 드래그 중 고스트 카드 및 컬럼 하이라이트 피드백 제공

#### F-06. 반응형 레이아웃
- 데스크탑 (≥ 768px): 3컬럼 수평 배치
- 태블릿 (768px~1023px): 간격 축소, 최소 너비 220px
- 모바일 (< 768px): 컬럼 세로 쌓기, 전체 너비

#### F-05. 데이터 영속성
- 카드 추가/삭제/이동 시 자동 저장
- 페이지 새로고침 후에도 데이터 유지
- **Phase 1**: localStorage 기반
- **Phase 2**: 백엔드 API + RDB 연동 (향후)

### 2.2 개선 기능 (Nice to Have)

| 기능 | 우선순위 |
|------|--------|
| 카드 내용 수정 (더블클릭 편집) | 중 |
| 같은 컬럼 내 카드 순서 변경 | 중 |
| 컬럼 헤더에 카드 수 배지 표시 | 하 |
| 다크모드 | 하 |
| 멀티 보드 (여러 보드 생성/전환) | 하 |
| 사용자 인증 및 개인 데이터 분리 | 하 |

## 3. 비기능 요구사항

| 항목 | 요구사항 |
|------|---------|
| 초기 로딩 | 1초 이내 (외부 리소스 없음) |
| 드래그 응답 | 지연 없이 즉시 반응 |
| 브라우저 호환 | 최신 Chrome, Firefox, Safari, Edge |
| 오프라인 동작 | localStorage 기반으로 오프라인에서도 동작 |
| 반응형 | 모바일(< 768px), 태블릿(768~1023px), 데스크탑(≥ 1024px) 지원 |
| 모바일 터치 | Touch Events API 기반 드래그 앤 드롭 지원 |
