# User Flow
# Kanban Board

---

## 1. 전체 흐름 개요

```
사용자 접속
    │
    ▼
localStorage 데이터 존재?
    ├── YES → 저장된 카드 복원
    └── NO  → 샘플 카드 5개 표시
    │
    ▼
보드 화면 (To-do / In-progress / Done)
    │
    ├── [카드 추가]
    ├── [카드 삭제]
    └── [카드 이동]
```

---

## 2. 카드 추가 흐름

```
컬럼 헤더의 [+] 버튼 클릭
    │
    ▼
prompt() 다이얼로그 표시
    │
    ├── 취소 or 빈 입력 → 아무 것도 하지 않음
    │
    └── 텍스트 입력 후 확인
            │
            ▼
        해당 컬럼 하단에 카드 생성
            │
            ▼
        localStorage 자동 저장
```

---

## 3. 카드 삭제 흐름

```
카드에 마우스 호버
    │
    ▼
카드 우측에 [×] 버튼 표시
    │
    ▼
[×] 버튼 클릭
    │
    ▼
카드 즉시 제거 (확인 없음)
    │
    ▼
localStorage 자동 저장
```

---

## 4. 카드 이동 흐름 (드래그 앤 드롭)

```
카드 드래그 시작
    │
    ▼
카드 반투명 처리 (.dragging)
    │
    ▼
다른 컬럼 위로 드래그
    │
    ▼
대상 컬럼 하이라이트 (.drag-over)
    │
    ├── 드롭 없이 취소 (ESC or 원래 위치 복귀)
    │       │
    │       ▼
    │   스타일 초기화, 위치 변경 없음
    │
    └── 대상 컬럼에 드롭
            │
            ▼
        카드가 대상 컬럼 하단으로 이동
            │
            ▼
        스타일 초기화
            │
            ▼
        localStorage 자동 저장
```

---

## 5. 페이지 재방문 흐름

```
브라우저에서 페이지 새로고침 또는 재방문
    │
    ▼
localStorage('kanban-board') 읽기
    │
    ├── 데이터 있음 → 저장된 카드 전부 복원 (컬럼별 순서 유지)
    │
    └── 데이터 없음 → 샘플 카드 5개로 초기화
```

---

## 6. Phase 2 — RDB 연동 후 흐름 (향후)

```
사용자 접속
    │
    ▼
로그인 (JWT 발급)
    │
    ▼
GET /api/boards/:id → 카드 목록 수신
    │
    ▼
보드 화면 렌더링
    │
    ├── 카드 추가 → POST /api/cards
    ├── 카드 삭제 → DELETE /api/cards/:id
    └── 카드 이동 → PATCH /api/cards/:id/move
```
